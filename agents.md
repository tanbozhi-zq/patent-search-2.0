# Agent Rules

1. 本项目当前按轻量流程推进，常设角色为项目总控、开发人员、联调/接入人员。各 Agent 需要先判断自己承担的角色，再按对应职责行动。

2. 项目总控负责维护项目边界、阶段计划、字段映射、API 设计、查询语法清单、Git 状态和交付文档；同时负责调度开发人员与联调/接入人员，并审查实现是否符合文档约定。

3. 开发人员负责按照项目总控确认的需求和文档实现后端服务、Tool、MCP 或部署相关代码，并在提交说明中记录本地自查结果。

4. 后续阶段不再单独设置测试环境、测试人员派工、测试验收单或测试报告；质量门由开发自查、项目总控 review、真实联调记录和交付文档复核承担。

5. 联调/接入人员只在真实 Flow / DeerFlow / MCP 接入阶段参与，负责提供接入配置、调用结果、问题现象和必要日志；不再产出独立测试环境报告。

6. 普通说明类文档可以使用中文命名，并确保文件名能准确反映文档内容；真正用于工程交付、部署、接口、联调等正式交付文档时，按照工程最佳实践和项目约定命名，不强制使用中文。

7. 需要时刻注意整个项目的文件管理，按照工程化上的最佳实践，保证整个项目的文件树条理清晰。
