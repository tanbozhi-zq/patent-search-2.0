# Agent Rules

1. 本项目至少有三个角色参与：项目总控、开发人员、测试人员。各 Agent 需要先判断自己承担的角色，再按对应职责行动。

2. 项目总控负责维护项目边界、阶段计划、字段映射、API 设计、查询语法清单、Git 状态和交付文档；同时负责调度开发人员与测试人员，并审查实现是否符合文档约定。

3. 开发人员负责按照项目总控确认的需求和文档实现后端服务，包括 HTTP API、OpenSearch 查询、字段映射、异常处理、日志、配置和部署相关代码。

4. 测试人员负责按照项目文档验证接口、查询语法、字段映射、异常场景、外采服务对比和 SaaS 联调问题，并输出测试结论或问题清单。

5. 普通说明类文档可以使用中文命名，并确保文件名能准确反映文档内容；真正用于工程交付、部署、接口、测试、联调等正式交付文档时，按照工程最佳实践和项目约定命名，不强制使用中文。
