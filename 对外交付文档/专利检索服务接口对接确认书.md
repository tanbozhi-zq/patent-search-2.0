# 专利检索服务接口对接确认书

## 1. 文档目的

本文用于在正式接入前，与需求方和接入方确认专利检索服务的交付形式、接口范围、查询能力、字段契约、错误响应、能力边界和上线验收标准。

确认完成后，服务提供方负责完成部署并向接入方提供：

- 服务访问地址 `BASE_URL`
- 接口调用凭证 `X-API-Key`
- 接口契约文档
- 查询语法说明
- 上线验收清单

## 2. 交付形式

本服务按“已部署 HTTP API 服务”形式交付。接入方不需要自行部署代码，只需要按约定调用服务接口。

| 项 | 约定 |
|---|---|
| 交付对象 | 专利检索 HTTP API 服务 |
| 调用方式 | 接入方通过 `BASE_URL` 访问 |
| 鉴权方式 | 请求头 `X-API-Key` |
| 文档交付 | 接口契约、查询语法、验收清单 |
| 部署责任 | 服务提供方负责部署、启动、日志和基础运维 |
| 接入责任 | 接入方负责保存 API Key，并按接口契约调用 |

## 3. 接口范围

| 接口 | 方法 | 路径 | 用途 |
|---|---|---|---|
| 健康检查 | `GET` | `/health` | 检查服务是否可用 |
| 专利检索 | `POST` | `/api/patent/search` | 按查询式检索专利 |
| 专利详情 | `GET` | `/api/patent/detail/{patent_id}` | 获取单件专利详情 |
| 引证信息 | `GET` | `/api/patent/citations/{patent_id}` | 获取引用、被引和原始引证字段 |

## 4. 查询能力

检索接口通过 `q` 参数表达查询条件。当前支持以下字段：

| 查询字段 | 含义 |
|---|---|
| `title` | 标题 |
| `ab` | 摘要 |
| `tscd` | 标题、摘要、首权、完整权利要求书、说明书综合检索 |
| `mainClaim` | 首权或主权利要求 |
| `claims` | 完整权利要求书 |
| `description` | 说明书 |
| `ipc` | IPC 分类 |
| `applicant` | 申请人 |
| `currentAssignee` | 当前权利人 |
| `legalStatus` | 法律状态 |
| `type` | 专利类型 |
| `ad` | 申请日范围 |
| `documentYear` | 公开年范围 |

常用示例：

```text
title:(阀门)
ab:(缓冲)
tscd:("均衡" OR "平衡")
mainClaim:(均衡)
claims:(均衡)
description:(均衡)
ipc:H02M AND claims:(均衡)
ad:[2020-01-01 TO 2020-12-31]
documentYear:[2020 TO 2024]
```

完整语法见 `专利检索服务查询语法说明.md`。

## 5. 核心接口契约

### 5.1 鉴权

除健康检查外，业务接口默认要求在请求头中传入：

```http
X-API-Key: <由服务提供方发放的 API Key>
```

鉴权失败返回 HTTP 401，错误码 `40101`。

### 5.2 分页

| 参数 | 约定 |
|---|---|
| `page` | 从 1 开始 |
| `page_size` | 默认 50，最大 100 |
| 非法页码 | HTTP 400，错误码 `40003` |

### 5.3 错误响应

错误响应统一为扁平结构，不包含外层 `detail`：

```json
{
  "success": false,
  "code": 40001,
  "message": "q 查询语法错误：字段 claims 的值不能为空",
  "data": null
}
```

### 5.4 字段兼容

检索和详情接口会同时保留常用 camelCase 字段与 snake_case 兼容字段，例如：

| camelCase | snake_case |
|---|---|
| `applicationNumber` | `application_number` |
| `documentNumber` | `document_number` |
| `applicationDate` | `application_date` |
| `documentDate` | `document_date` |
| `legalStatus` | `legal_status` |
| `mainIpc` | `main_ipc` |

## 6. 需确认的能力边界

| 边界项 | 当前约定 |
|---|---|
| 高亮 | `highlight=1` 兼容接收，但当前不返回高亮片段 |
| 说明书正文 | 详情接口仅在 `include_description=true` 时返回 `description` 字段 |
| 说明书检索 | `description:(关键词)` 用于检索说明书字段，不等同于详情默认返回说明书正文 |
| 引证信息 | 对结构化引证字段尽力归一化，同时保留原始字段 |
| 数据质量 | 接口按现有 OpenSearch 数据返回，不在本次交付中修复源数据质量 |
| 索引结构 | 不修改 OpenSearch mapping，不重建索引 |
| 前端页面 | 本次交付对象为 HTTP API，不包含生产前端页面 |
| 外部 PatentHub | 外部 PatentHub 线上 Token 验证不作为本服务交付前置条件 |

## 7. 上线验收标准

部署完成后，以下项目通过即可视为接口具备接入条件：

| 编号 | 验收项 | 预期 |
|---|---|---|
| 1 | 健康检查 | `/health` 返回 HTTP 200，状态为 healthy |
| 2 | 基础检索 | `/api/patent/search` 返回 `total/page/page_size/records` |
| 3 | 字段检索 | `title`、`ab`、`tscd` 可正常检索 |
| 4 | 细粒度文本检索 | `mainClaim`、`claims`、`description` 可正常检索 |
| 5 | 详情接口 | 返回专利核心字段，`include_description=true` 时返回说明书 |
| 6 | 引证接口 | 返回 `cited_by`、`patent_references` 和原始兼容字段 |
| 7 | 错误码 | 非法查询返回 HTTP 400，错误码 `40001` |
| 8 | 鉴权 | 缺失或错误 API Key 返回 HTTP 401，错误码 `40101` |

详细验收步骤见 `专利检索服务上线验收清单.md`。
