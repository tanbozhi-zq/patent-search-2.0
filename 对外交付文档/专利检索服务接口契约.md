# 专利检索服务接口契约

## 1. 基础约定

| 项 | 说明 |
|---|---|
| Base URL | 由服务提供方部署后提供，例如 `http://host:8000` 或网关地址 |
| 请求格式 | JSON |
| 响应格式 | JSON |
| 鉴权 Header | `X-API-Key: <API Key>` |
| Content-Type | `application/json` |
| 字符集 | UTF-8 |

业务接口默认需要鉴权。健康检查接口可不传 `X-API-Key`。

## 2. 通用错误结构

错误响应统一为：

```json
{
  "success": false,
  "code": 40001,
  "message": "q 查询语法错误：缺少右括号",
  "data": null
}
```

错误码：

| code | HTTP | 含义 |
|---:|---:|---|
| `40001` | 400 | 查询语法错误 |
| `40002` | 400 | 普通参数非法 |
| `40003` | 400 | `page` 或 `page_size` 非法 |
| `40101` | 401 | 鉴权缺失或错误 |
| `40401` | 404 | 专利不存在 |
| `50001` | 502 | OpenSearch 查询异常 |
| `50002` | 500 | 服务内部异常 |

## 3. 健康检查

```http
GET /health
```

响应示例：

```json
{
  "success": true,
  "code": 0,
  "message": "ok",
  "data": {
    "status": "healthy",
    "service": "patent-search-service"
  }
}
```

## 4. 专利检索

```http
POST /api/patent/search
```

### 4.1 请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `q` | string | 是 | 无 | 查询式，最大 1000 字符 |
| `ds` | string | 否 | `cn` | 数据范围，支持 `cn`、`all` |
| `sort` | string | 否 | `relation` | 排序，支持 `relation`、`!applicationDate` |
| `page` | integer | 否 | `1` | 页码，从 1 开始 |
| `page_size` | integer | 否 | `50` | 每页数量，最大 100 |
| `highlight` | integer | 否 | `0` | 支持 `0`、`1`；当前仅兼容接收 |
| `index_analyzer_mode` | string | 否 | `compat` | 支持 `compat`、`normal` |

`q` 支持字段查询、范围查询、`AND`、`OR`、`NOT` 和常规多级括号分组；不承诺支持极端深度嵌套或明显不可读的超长表达式。完整规则见 `专利检索服务查询语法说明.md`。

### 4.2 请求示例

```bash
curl -X POST "$BASE_URL/api/patent/search" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $API_TOKEN" \
  -d '{
    "q": "ipc:H02M AND (tscd:(\"均衡\" OR \"平衡\") OR mainClaim:(均衡))",
    "ds": "cn",
    "sort": "relation",
    "page": 1,
    "page_size": 10,
    "highlight": 0
  }'
```

### 4.4 检索字段说明

| 字段 | 说明 |
|---|---|
| `records` | 当前 HTTP API 的检索结果列表字段 |

| `application_number` | `applicationNumber` 的 snake_case 兼容字段 |

| `document_date` | `documentDate` 的 snake_case 兼容字段 |
| `legal_status` | `legalStatus` 的 snake_case 兼容字段 |
| `main_ipc` | `mainIpc` 的 snake_case 兼容字段 |

`highlight=1` 不报错，但当前响应不返回高亮片段。

## 5. 专利详情

```http
GET /api/patent/detail/{patent_id}
```

### 5.1 参数

| 参数 | 位置 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|---|
| `patent_id` | path | string | 是 | 无 | 检索结果中的专利 ID |
| `include_description` | query | boolean | 否 | `false` | 是否返回说明书正文 |

### 5.2 请求示例

```bash
curl "$BASE_URL/api/patent/detail/$PATENT_ID?include_description=true" \
  -H "X-API-Key: $API_TOKEN"
```

## 6. 引证信息

```http
GET /api/patent/citations/{patent_id}
```

### 6.1 请求示例

```bash
curl "$BASE_URL/api/patent/citations/$PATENT_ID" \
  -H "X-API-Key: $API_TOKEN"
```

### 6.2 成功响应

```json
{
  "patent_id": "cn-xxx",
  "cited_by": [],
  "patent_references": [
    {
      "id": "CN112501955A",
      "title": "",
      "applicant": "",
      "application_date": "2021-03-16",
      "application_number": "",
      "type": "",
      "legal_status": "",
      "main_ipc": ""
    }
  ],
  "non_patent_references": [],
  "referencesCited": [],
  "referencesCitedRaw": "",
  "referencesCitedText": "",
  "relatedDocuments": []
}
```

### 6.3 字段说明

| 字段 | 说明 |
|---|---|
| `cited_by` | 被引专利摘要列表，尽力从原始字段归一化 |
| `patent_references` | 引用专利摘要列表，尽力从原始字段归一化 |
| `non_patent_references` | 非专利文献或原始引用文本 |
| `referencesCited` | OpenSearch 原始结构化引证字段 |
| `referencesCitedRaw` | OpenSearch 原始引证文本 |
| `referencesCitedText` | OpenSearch 文本化引证列表 |
| `relatedDocuments` | OpenSearch 原始相关文献 |

归一化摘要字段：

| 字段 | 说明 |
|---|---|
| `id` | 引用专利 ID；可由 `Country + DocNumber + Kind` 合成 |
| `title` | 标题 |
| `applicant` | 申请人 |
| `application_date` | 申请日；缺少 `ApplicationDate` 时可使用 `Date` |
| `application_number` | 申请号 |
| `type` | 专利类型 |
| `legal_status` | 法律状态 |
| `main_ipc` | 主 IPC |

## 7. 分页与参数错误

| 场景 | HTTP | code |
|---|---:|---:|
| `page < 1` | 400 | `40003` |
| `page_size < 1` | 400 | `40003` |
| `page_size > 100` | 400 | `40003` |
| `ds` 非 `cn/all` | 400 | `40002` |
| `sort` 非 `relation/!applicationDate` | 400 | `40002` |
| `highlight` 非 `0/1` | 400 | `40002` |
| `q` 语法非法 | 400 | `40001` |
