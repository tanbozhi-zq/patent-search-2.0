# 阶段 11 上线交付计划

## 1. 当前判断

阶段 10.5 已关闭，核心接口能力已经完成。阶段 11 不直接从服务器部署开始，而是先完成对外接口确认，再进入部署执行和上线验收。

本阶段面向的最终交付形式是：

```text
服务提供方部署 HTTP API 服务，向需求方/接入方交付 Base URL、API Key、接口文档、查询语法文档和验收清单。
```

## 2. 文档分层

对外文档不出现内部阶段编号、派工、角色分工和 Git 管理信息，只描述接口契约、能力边界和验收标准。

内部文档用于项目总控、开发、测试和运维协作，可以保留阶段编号和内部过程。

| 类型 | 文件 | 用途 |
|---|---|---|
| 对外 | `对外交付文档/专利检索服务接口对接确认书.md` | 给需求方确认交付形式、接口范围和边界 |
| 对外 | `对外交付文档/专利检索服务接口契约.md` | 给接入开发按契约调用 |
| 对外 | `对外交付文档/专利检索服务查询语法说明.md` | 给产品、运营和开发理解 q 查询语法 |
| 对外 | `对外交付文档/专利检索服务上线验收清单.md` | 给需求方和接入方做上线验收 |
| 内部 | `docs/ops/deployment_runbook.md` | 给部署和运维执行 |
| 内部 | `docs/internal/stage11_delivery_plan.md` | 给项目总控维护节奏和边界 |

## 3. 阶段 11 工作顺序

1. 完成对外文档初稿。
2. 与需求方确认交付形式和接口契约。
3. 根据确认意见修订文档。
4. 冻结上线版本提交号。
5. 按部署手册部署服务。
6. 执行部署后 smoke。
7. 使用上线验收清单和需求方共同确认。
8. 输出部署报告和运维交接记录。

## 4. 需求方确认重点

- 是否接受“已部署 HTTP API 服务 + Base URL + API Key + 文档”的交付形式。
- 接口路径、请求参数、响应字段是否满足接入需求。
- 查询语法是否覆盖当前产品需求。
- 错误码和错误结构是否满足接入方处理要求。
- `highlight=1` 当前不返回高亮片段是否可接受。
- 当前不修改 OpenSearch mapping 是否可接受。
- 上线验收清单是否可作为放行标准。

## 5. 部署前置条件

部署前必须确认：

- 对外接口确认书已被需求方接受。
- 生产或准生产 Base URL 形式已确定。
- `API_TOKEN` 已生成并通过安全渠道交付。
- OpenSearch 账号、密码、索引和连通性已确认。
- systemd 部署目录、日志目录和服务端口已确认。
- 回滚方案已明确。

## 6. 阶段边界

阶段 11 做：

- 对外接口文档整理。
- 对外查询语法和验收清单整理。
- 部署手册整理。
- 目标环境部署。
- 部署后 smoke。
- 验收记录和运维交接。

阶段 11 不做：

- 不修改 OpenSearch mapping。
- 不重建索引。
- 不新增前端生产页面。
- 不新增高亮片段返回能力。
- 不修复源数据质量问题。
- 不提交任何密钥。
- 不直接扩大到未确认的生产流量。

## 7. 完成标准

阶段 11 完成需要同时满足：

- 对外文档已确认。
- 版本提交号已记录。
- 服务部署成功。
- health、search、detail、citations smoke 通过。
- 鉴权和错误码验证通过。
- 需求方验收清单签收或明确确认。
- 部署报告和运维交接文档已归档。
